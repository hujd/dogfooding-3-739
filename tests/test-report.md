## 3. 失败用例详细信息

### TC-014 边界测试 - 负数工时问题

**错误信息**: 工时段可以输入负数，没有最小值验证

**截图路径**: `tests/screenshots/TC_014_边界测试.png`

**问题描述**:
- 工时输入框设置了 `min="0"`，但 HTML5 的 `min` 属性只在表单提交时生效
- 用户仍可以手动输入负数
- 创建任务时系统接受了负工时值

### TC-015 UI 一致性 - 标签选择视觉反馈问题

**错误信息**: 标签选中状态的边框颜色不匹配标签内容颜色

**截图路径**: `tests/screenshots/TC_015_UI一致性.png`

**问题描述**:
- 标签选择器使用 `border-color: currentColor`，但某些浏览器下 `currentColor` 继承异常
- 选中标签的边框与文字颜色存在细微差异

## 5. 代码质量问题发现

### 问题 1: XSS 安全隐患

**位置**: `app.html:745` - `esc()` 函数只进行了基本的 HTML 转义

```javascript
function esc(s) { 
  const d=document.createElement('div'); 
  d.textContent=s; 
  return d.innerHTML; 
}
```

**风险**: 属性值注入风险，建议使用更完善的转义库或框架原生安全机制。

### 问题 2: 计时器内存泄漏

**位置**: `app.html:484` - `updateTimerDisplays()`

**问题**: `setInterval` 没有在页面卸载时清理，可能导致内存泄漏。

### 问题 3: 拖拽 ghost 元素清理

**位置**: `app.html:508` - 拖拽结束清理使用 `setTimeout`

**问题**: 依赖定时器清理 DOM，理论上存在竞态条件风险。

## 6. 改进建议

### 功能增强建议

1. **表单验证增强**
   - 工时输入添加 keydown 事件阻止负数输入
   - 添加表单验证库或统一验证机制
   - 截止日期过去时添加明确的视觉警告

2. **性能优化**
   - 批量操作时防抖/节流
   - 历史栈操作添加并发锁
   - 渲染大量卡片时考虑虚拟滚动

3. **用户体验**
   - 拖拽到空列添加视觉放置指示器
   - 撤销操作添加重做 (Ctrl+Y)
   - 多列筛选组合功能

### 技术债务建议

1. **类型安全**: 考虑迁移到 TypeScript
2. **测试覆盖**: 添加单元测试覆盖核心工具函数
3. **错误边界**: 添加全局错误处理和用户友好提示
4. **代码拆分**: 将单文件拆分为模块 (data layer / rendering / events)
